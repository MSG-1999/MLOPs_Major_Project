Large-scale latent diffusion models excel at broad visual generation вАФ but they systematically under-represent **culturally specific imagery**. Generic prompts to Stable Diffusion v1.5 fail to reproduce the warmth of Diwali diyas, the chromatic chaos of Holi, or the grandeur of Ganesh Chaturthi processions.

The system is designed from first principles for **reproducibility**, **extensibility**, and **production readiness** вАФ not just research demos.

### ModelManager

- **Singleton pattern** with `threading.Lock` вАФ one model load per process
- Device resolution cascade: `CUDA вЖТ Apple MPS вЖТ CPU`
- Three-level fallback during `from_pretrained` for models lacking fp16 safetensors
- **Scheduler hot-swapping** across DDIM, Euler-A, DPM++ 2M, PNDM, LMS, DPM++ SDE

### InferenceEngine

- Accepts `GenerationConfig` dataclass; always returns `GenerationResult`
- Explicit CUDA OOM handling with `torch.cuda.empty_cache()` and actionable error messages
- The UI layer **never receives an unhandled exception**

### Dataset: IndianFestivals

DreamForge fine-tunes on the [`AIMLOps-C4-G16/IndianFestivals`](https://huggingface.co/datasets/AIMLOps-C4-G16/IndianFestivals) dataset вАФ **3,500+ images** spanning seven festival categories with richly engineered text prompts:

| рЯЗЃрЯЗ≥ **Independence Day** | National flag, patriotic parade, saffronвАУwhiteвАУgreen |

### LoRA Formulation

LoRA injects trainable rank-decomposed matrices into the UNet attention layers while freezing the 860 M-parameter backbone:

```
h = WвВАx + (ќ±/r) ¬Ј B¬ЈA¬Јx

where:
  WвВА  вИИ вДЭбµИЋ£бµП  вАФ frozen pre-trained weight
  A   вИИ вДЭ ≥Ћ£бµП  вАФ initialised from рЭТ©(0, ѕГ¬≤)
  B   вИИ вДЭбµИЋ£ ≥  вАФ initialised to zero (zero-delta init)
  r           вАФ rank (controls adapter capacity)
  ќ±           вАФ scaling factor
```

**Target layers** (6 per UNet transformer block):
`to_q ¬Ј to_k ¬Ј to_v ¬Ј to_out.0 ¬Ј ff.net.0.proj ¬Ј ff.net.2`

### Best Trial Configuration

| Parameter | Value | Notes |
|---|---|---|
| LoRA Rank `r` | **16** | Also evaluated: 4, 8 |
| LoRA Alpha `ќ±` | **32** | Scale = ќ±/r = 2.0 |

### Training Objective

The denoising score-matching loss in latent space:

```
вДТ = рЭФЉ[вАЦќµ вИТ ќµќЄ(zвВЬ, t, ѕДќЄ(c))вАЦ¬≤]

where:
  zвВА = E(x)    вАФ VAE-encoded latent
  ќµ ~ рЭТ©(0, I)  вАФ ground-truth noise
  ѕДќЄ(c)        вАФ CLIP text embedding (frozen)
```

The VAE encoder, VAE decoder, and CLIP text encoder are **all frozen**. Only LoRA parameters inside the UNet are updated.

## вЪ° ASHA Hyperparameter Optimisation

### Algorithm

Asynchronous Successive Halving (ASHA) launches many trials concurrently and **early-stops unpromising ones** вАФ eliminating the synchronisation barrier of standard SHA. A trial is promoted to the next rung only if it ranks in the top `1/ќЈ` fraction of all trials reaching that rung.

### Search Space

```python
r       вИИ  {4, 8, 16}
ќ±       вИИ  {16, 32, 64}
p_drop  ~  Uniform(0.0, 0.2)
ќЈ_lr    ~  LogUniform(1e-5, 2e-4)
B       вИИ  {1, 2, 4}
```

### Integration

```python
# Non-invasive loop integration
if trial is not None:
    trial.report(train_loss, global_step)
    if trial.should_prune():
        raise optuna.TrialPruned()
```

## вЬВпЄП Post-Training Pruning

DreamForge applies **вДУвВБ-norm magnitude pruning** to LoRA adapter matrices A and B after training. The frozen backbone is never touched.

```python
import torch.nn.utils.prune as prune

def prune_lora_weights(unet, sparsity=0.20):
    lora_layers = [
        (m, 'weight') for n, m in unet.named_modules()
        if 'lora' in n.lower() and hasattr(m, 'weight')
    ]
    prune.global_unstructured(
        lora_layers,
        pruning_method=prune.L1Unstructured,
        amount=sparsity,
    )
    # Make masks permanent before saving
    for module, param in lora_layers:
        prune.remove(module, param)
    return unet
```

### SparsityвАУQuality Trade-off

| Sparsity | Val Loss | Adapter Size Reduction | Recommendation |
|:---:|:---:|:---:|:---:|
| 0% | 0.1823 | вАФ | Baseline |
| 10% | 0.1829 | 9% | Conservative |
| **20%** | **0.1847** | **18%** | вЬЕ **Recommended** |
| 30% | 0.1891 | 26% | Acceptable |
| 50% | 0.2103 | 40% | Degraded |
| 70% | 0.2614 | 57% | вЭМ Not recommended |

> **20% sparsity is the sweet spot**: 18% memory reduction for only a 1.3% relative increase in validation loss.
